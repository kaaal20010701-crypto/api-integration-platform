# 厂商 API 调用 Skill · 通用参考（reference）

---

## 异步轮询通用模板

1. 调用“创建任务”接口，从响应中取 `task_id`（或文档中约定的字段）。
2. 循环：间隔 N 秒（按厂商建议，如 5～10 秒）调用“获取结果”接口，传入 `task_id`。
3. 若返回表示“处理中”的状态（如 Pending、processing、error_code: 1），**继续轮询**，不要当作最终错误返回给前端。
4. 若返回“成功”或“失败”终态，退出循环，将结果或错误返回给前端。
5. 设置最大轮询次数或总时长，避免死循环。

---

## 错误码处理

- 鉴权失败：Token 接口返回非 200 或无 access_token，应报错并提示检查 Key/Secret。
- 业务错误：厂商返回的 error_code / error_msg 可原样或整理后通过 `jsonify(…), 400` 返回，便于前端展示。

---

## 新增厂商时的书写格式

在本 skill 下新建 `厂商英文名.md`（如 `aliyun.md`），结构建议：

1. **鉴权**：方式、URL、参数、如何携带到业务请求。
2. **具体接口**：URL、方法、请求体、同步/异步。
3. **若为异步**：创建任务与获取结果的 URL 与 body；轮询间隔；“处理中”与“成功/失败”的判定方式。
4. **注意点**：如 base64 是否 urlencode、图片格式要求等。

最后在 [SKILL.md](SKILL.md) 的厂商列表中增加该厂商及对应文件链接。
