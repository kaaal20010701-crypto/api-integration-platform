# 可视化 Skill · 实现参考（reference）

---

## 左侧 JSON 区

- **展示**：完整 JSON，`JSON.stringify(data, null, 2)` 格式化，`pre` 或 `code` 显示。
- **一键复制**：`navigator.clipboard.writeText(JSON.stringify(data, null, 2))`，可配“已复制”短暂提示。
- **一键下载**：构造 `Blob`，用 `URL.createObjectURL` + `<a download="xxx.json">` 触发下载，文件名建议带时间戳如 `result_YYYYMMDDHHmmss.json`。
- **位置**：与面板标题同一行，按钮放标题右侧（如 `panel-actions` 内）。

---

## 布局结构

- 结果区外层：`result-layout`（flex 或 grid），左 `result-json`，右 `result-vis`。
- JSON 区：`result-json` 内正常显示 JSON。
- 可视化区：`result-vis` 内放置基于输入图的可视化内容（见 [ocr-type.md](ocr-type.md)）。
- 错误时：左侧保留完整 JSON；右侧框内可显示错误提示或留空。
- **版式元素文本区域**：当文本区域无法显示全部内容时，需出现滚动条（`overflow-y: auto`），确保内容能够被完整查看，禁止使用 `overflow: hidden` 截断。

---

## 原图标注（右侧可视化实现）

- **要求**：将 JSON 中的坐标、区域等信息叠加到**用户上传的输入图片**上，便于快速定位。输入图需在提交时保留（如 `URL.createObjectURL(file)` 或后端返回的图片 URL）。
- **布局**：原图容器 `position: relative`，内放 `<img>` 和一层 `position: absolute` 的 overlay，overlay 内用绝对定位的 div 画框/标签。
- **坐标换算**：用 `img.naturalWidth/naturalHeight` 与显示尺寸 `offsetWidth/offsetHeight` 求比例，将接口坐标（原图像素）换算到显示尺寸。
- **下载标注图**：可用 canvas 绘制原图并按同一坐标画框，`toDataURL('image/png')` 后触发下载；若跨域导致 canvas 污染，可回退为下载原图。

## 当用户对首版可视化不满意时

当用户对首次生成的右侧可视化不满意，并额外提供：

- 来自**官方技术文档**中的「返回参数说明」（字段表），以及
- 实际调用得到的 **JSON 返回示例**（可能很长），

则可视化设计与实现必须满足以下附加要求：

1. **同时参考技术文档与 JSON**  
   - 不允许只看 JSON 或只看字段表，必须**并行使用**：  
     - 用技术文档字段表确认每个字段的语义、取值范围、是否必选；  
     - 用实际 JSON 验证真实返回结构（嵌套层级、数组/对象形态、典型组合）。
2. **基于字段语义重新设计可视化**  
   - 逐项对照字段表里的关键字段（如坐标、题目类型、答案、配图、批改得分等）在原图进行可视化。
     -如果需要在原图上显示文本时，需要避免和打框的颜色相同，用不同的颜色区分文本和框

3. **全面理解长 JSON，而非只截取片段**  
   - 即使 JSON 很长，也要结合字段表，理解整体结构和各字段关系；  
   - 至少选取一两条**代表性样例**（包含多种题型或元素类型）来对照设计，而不是只基于最简单场景。
4. **新的设计必须真正落实到前端代码**  
   - 在对应 `demos/*.html` 中直接实现新的设计：  
     - 更新右侧 DOM 结构与样式（如题目列表区、标签、交互按钮）；  
     - 更新 JS 解析逻辑，使字段映射与技术文档/JSON 一致；  
     - 确保交互（如点击列表高亮原图、下载标注图等）与新的可视化方案匹配。
---

## checklist

只要有改动会影响前端展示（HTML / CSS / JS），
**在前端代码中，在改动前后对照本文件做自查**，自查时至少确认：

1. 确认仍然是「左 JSON / 右可视化」的单套布局；
2. 确认仍然提供 JSON 的一键复制 / 一键下载能力；
3. 不要在同一个 `demos/*.html` 里出现第二份 `<!DOCTYPE html>...<html>...</html>` 或重复的一整套结果区结构；
4. 需要在原图上正确叠加可视化（对 OCR 类）。
