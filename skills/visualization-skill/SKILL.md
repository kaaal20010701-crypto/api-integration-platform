# 可视化 Skill（API 结果展示设计）

本 skill 约定 Demo 页结果区的**基本布局**与**右侧可视化通用原则**。**集成 skill** 在实现新 Demo 页时，按本约定实现。



## 四部分文件说明

| 文件 | 用途 |
|------|------|
| **SKILL.md**（本文件） | 主入口：通用约定、右侧可视化原则。 |
| **[ocr-type.md](ocr-type.md)** | OCR 类：右侧基于输入图的可视化流程。 |
| **[other-types.md](other-types.md)** | 其他类型：左侧 JSON；右侧留框。 |
| **[reference.md](reference.md)** | 左侧 JSON 操作、布局结构、原图标注实现。 |

## 通用约定

- **布局**：结果区**左右分栏**；左侧“原始数据”（JSON），右侧“可视化区”。
- **左侧**：JSON **正常显示**，完整展示；提供**一键复制**（复制 JSON 到剪贴板）和**一键下载**（下载 `.json` 文件）。详见 [reference.md](reference.md)。
- **滚动**：demo 页 body 支持纵向滚动；可提供“回到顶部”按钮。
- **错误**：接口错误时，左侧保留原始 JSON；右侧框内可简单展示错误提示或留空。
- **单页结构**：每个 `demos/*.html` Demo 页内部只保留**一套**完整页面结构与一套“左 JSON / 右可视化”的结果布局；当需要迭代 UI 或交互时，应在**原有文件内就地修改对应片段**，而不是在文件末尾再追加第二份 `<!DOCTYPE html>...<html>...</html>` 或重复的一整套右侧可视化表单，以避免出现左右两份几乎相同的 Demo 画面。

## 右侧可视化通用原则（以 OCR 为主）

因大部分业务场景为 OCR，右侧可视化按以下流程设计：

1. **理解输出结构**：阅读技术文档或任务提供的 JSON 文档，理解接口输出结构及包含的信息（如坐标、题干、选项、批改结果、切题区域等）。
2. **在原图上可视化**：结合 JSON 中已有的信息（坐标、识别的文字内容等），在**输入图片**上进行可视化（如叠加框、标签、高亮区域），便于用户快速定位与理解。
3. **放入右侧**：将上述可视化内容呈现在右侧区域。

详见 [ocr-type.md](ocr-type.md) 与 [reference.md](reference.md) § 原图标注。
